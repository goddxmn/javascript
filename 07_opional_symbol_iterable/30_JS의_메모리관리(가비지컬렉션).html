/* 가비지 컬렉터 - 자바스크립트는 객체가 생성되었을 때 자동으로 메모리를 할당하고 더 이상 필요하지 않을 때 자동으로 해제한다. 메모리 생존주기 메모리 생존주기는 프로그래밍 언어와 관계없이 비슷하다. 필요할 때 할당한다. - 예)변수를 선언할 때 자동으로 메모리를 할당 할당된 메모리를 사용한다. (읽기, 쓰기) 더 이상 필요하지 않으면 해제한다. 자바스크립트와 같은 고수준 언어들은 "가비지 콜렉터(GC)"이라는 자동 메모리 관리 방법을 사용한다. 가비지 콜렉터의 목적은 메모리 할당을 추적하고 할당된 메모리 블록이 더 이상 필요하지 않게 되었는지를 판단하여 회수하는 것이다. */ let n = 123; // 숫자 타입에 맞게 메모리 자동 할당 let s = 'Inyong' // 문자열 타입에 맞게 메모리 자동 할당 let o = { a:1, b:null }; // 오브젝트와 해당 오브젝트 내에 포함된 값들을 담기 위한 메모리 할당 let a = [1, null, 'abcd'] // 배열과 배열에 담긴 값들을 위한 메모리 할당 function f(a){ // 함수를 위한 할당 (함수는 호출 가능한 오브젝트이다) return a + 2; } //함수 호출을 통한 할당 //메서드의 반환 값에 의해 새로운 변수에 할당하면 메모리 할당이 일어난다. //뿐만 아니라 DOM Element를 동적으로 생성하기 위해 'document.createElement' 함수를 이용할 때도 새롭게 메모리를 할당하게 된다. const d = new Date(); // Date 개체를 위해 메모리를 할당 const $div = document.createElement('div'); // DOM 엘리먼트를 위해 메모리를 할당한다.
